CSS技巧

09/03/2016 CSS

尽量减少代码的重复

  • 代码可维护性的最大要素是尽量减少改动时要编辑的地方。

  • 当某些值相互依赖时，我们应该把他们的相互关系用代码表达出来。

      // 优
  .font{
      font-size: 20px;
      line-height: 1.5;
  }
  // 劣
  .font{
      font-size: 20px;
      line-height: 30px;  
  }

  • 代码易维护和代码精简，两者不可兼得，根据实际情况选择。

      // 易维护
  .box{
      border-width: 10px;
      border-left: 0;
  }
  // 精简，但如果border宽度改为20px时，需要改动三个值
  .box{
      border-width: 10px 10px 10px 0;
  }

  • 善用继承 inherit ，减少代码量。

      .box{
      color: #f00;
  }
  .box a{
      color: inherit;
  }

  • 每个媒体查询都会增加成本，你添加的媒体查询越多，你的CSS代码就会变得越来越经不起折腾。总的来说，响应式网页设计时，我们应尽最大努力实现弹性可伸缩的布局,并在媒体查询的各个断点区间内指定相应的尺寸。下面列举一些可以替代媒体查询的技巧：

    • 1.使用百分比长度来取代固定长度；
    • 2.当你需要在较大的分辨率下得到固定宽度时，使用 max-width 而不是 width ，因为它可以适应较小的分辨率，而无需使用媒体查询。
    • 3.不要忘记为替换元素（如：img、object、video、iframe等）设置一个 max-width: 100%;。
    • 4.假如背景图片需要完整的铺满一个容器，不管容器的尺寸如何变化，background-size: cover;这个属性都可以做到。
    • 5.但图片（或者其他元素）以行列式进行布局时，让视口的宽度来决定列的数量。弹性盒布局（即flexbox）或者 display: inline-block; 加上常规的文本折行行为，都可以实现这一点。 ==注意：UC浏览器不支持弹性盒，如果项目需要适配UC的话，还是换个方法实现吧。==
    • 6.在使用多列文本时,指定column-width(列宽)而不是指定 column-count(列数),这样它就可以在较小的屏幕上自动显示为单 列布局。

Tips：如果你发现自己需要一大堆媒体查询才能让设计适应大大小小的屏幕, 那么不妨后退一步,重新审视你的代码结构。因为在所有的情况下,响应式 都不是唯一需要考虑的问题。

  • 合理使用简写。两者各有优缺点，应视情况决定。

      // 简写属性，此时，背景一定是红色
  .box{
      background: #f00;
  }
  // 展开式属性，此时，背景可能会被background-image等属性破坏
  .box{
      background-color: #f00;
  }

  • 关于CSS预处理器（比如：Stylus、SASS、LESS等），他们为CSS的编写提供了一些便利，比如变量、mixin、函数、嵌套、颜色处理等，使用得当的话可以让代码更加灵活，但也可能使代码变得庞大、维护性差。并且很有可能会被滥用，比如无限嵌套，无疑降低了代码的运行速度。

  • RGBA（红Red、绿Green、蓝Blue、透明度Alpha）和HSLA（色调Hue、饱和度Saturation、亮度Lightness、透明度Alpha）。

  • box-shadow的叠加，实现多重边框的效果。

      .box{
      width: 100px;
      height: 100px;
      border: 10px solid rgba(0, 0, 0, 0.5);
      box-shadow: 0 0 0 10px #655, 
                  0 0 0 20px green,
                  0 2px 5px 30px hsla(360, 100%, 50%, 0.5);
  }

  • outline，描边。outline和border的区别在于outline不影响布局。有一段调试页面的代码如下，实现思路就是给每个元素添加outline属性。

      [].forEach.call($$("*"),function(a){a.style.outline="1px solid #"+(~~(Math.random()*(1<<24))).toString(16)})

  • calc() 函数内部的运算符的两侧各加一个空格。否则会产生解析错误，这个规则是为了向前兼容，未来在 calc() 内部可能会允许使用关键字。

  • 边框内圆角效果：

      .box {
      width: 100px;
      height: 50px;
      border-radius: 0.8em;
      padding: 1em;
      background: tan;
      outline: 0.6em solid #655;
      box-shadow: 0 0 0 .3em #655;
  }

  • 渐变（IE10+支持）。 这个属性有很多玩法，自行百度吧。 默认从上到下渐变，即从#f00到#000渐变。可以通过设置 linear-gradient() 的第一个参数，共有以下选项：

    • to top
    • to bottom // 默认
    • to right
    • to left

    • 接受一个角度，例如： 45deg

      .box{
  background: linear-gradient(#f00, #000);
}
