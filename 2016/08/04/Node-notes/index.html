NodeJS笔记

2016-08-04
Node.js

基本概念

  • NodeJS是一个JS脚本解析器，任何操作系统下安装NodeJS本质上做的事情都是把NodeJS执行程序复制到一个目录，然后保证这个目录在系统PATH环境变量下，以便于终端访问。
  • 终端下直接属兔node命令可进入命令交互模式，很适合用来测试一些JS代码片段，比如正则表达式。
  • NodeJS遵循CMD模块规范，主模块作为程序入口点，所有模块在执行过程中只初始化一次。
  • 除非JS模块不能满足需求，否则不要轻易使用二进制模块。

package.json

关于package.json的官方解释：package.json。

需要注意的地方：

  • package.json存储的内容必须是真正的json，而不是js对象。
  • name和version是package.json中最重要的字段，他们有一定的规则。
    • name应该简短、意义清晰；
    • 查看可用的名字：点击访问；
    • 查看可用的version范围：点击访问。
  • description：当前包的描述，当被搜索到是会显示；
  • keywords：存放简介、关键字；
  • homepage：当前项目的官方地址；
  • bugs：当前项目遇到的问题及解决方法，差不多类似于下面这样：
      { 
      "url" : "http://github.com/owner/project/issues", 
      "email" : "project@hostname.com"
  }
  • package.json中不能写注释。
  • author - 包的作者姓名。
  • contributors - 包的其他贡献者姓名。
  • dependencies - 依赖包列表。如果依赖包没有安装，npm 会自动将依赖包安装在 node_module 目录下。
  • repository - 包代码存放的地方的类型，可以是 git 或 svn，git 可在 Github 上。
  • main - main 字段是一个模块ID，它是一个指向你程序的主要项目。就是说，如果你包的名字叫 express，然后用户安装它，然后require(“express”)。
  • keywords - 关键字
